# AI Assistant Skills / AIアシスタントスキル

## Core Skills / コアスキル

### Programming Languages / プログラミング言語

The AI assistant should be proficient in:

AIアシスタントは以下に精通している必要があります：

- **Web Development / Web開発**
  - JavaScript/TypeScript
  - HTML/CSS
  - React, Vue, Angular
  - Node.js

- **Backend Development / バックエンド開発**
  - Python
  - Java
  - Go
  - Rust

- **Mobile Development / モバイル開発**
  - Swift (iOS)
  - Kotlin (Android)
  - React Native
  - Flutter

### Frameworks & Tools / フレームワークとツール

- **Version Control / バージョン管理**
  - Git (branching, merging, rebasing)
  - GitHub workflow

- **Testing / テスト**
  - Unit testing
  - Integration testing
  - End-to-end testing
  - Test-driven development (TDD)

- **Build Tools / ビルドツール**
  - npm/yarn/pnpm
  - webpack/vite
  - Maven/Gradle

- **CI/CD**
  - GitHub Actions
  - GitLab CI
  - Jenkins

### Architecture & Design / アーキテクチャと設計

- **Design Patterns / デザインパターン**
  - SOLID principles
  - MVC/MVVM
  - Repository pattern
  - Factory pattern

- **API Design / API設計**
  - RESTful APIs
  - GraphQL
  - gRPC
  - WebSocket

- **Database / データベース**
  - SQL (PostgreSQL, MySQL)
  - NoSQL (MongoDB, Redis)
  - ORM/ODM

## Required Capabilities / 必要な能力

### Code Review / コードレビュー

- Identify potential bugs and issues
- Suggest improvements and optimizations
- Check for security vulnerabilities
- Ensure code follows best practices
- 潜在的なバグと問題を特定する
- 改善と最適化を提案する
- セキュリティ脆弱性をチェックする
- コードがベストプラクティスに従っていることを確認する

### Documentation / ドキュメンテーション

- Write clear and comprehensive documentation
- Create API documentation
- Generate code comments
- Maintain README files
- 明確で包括的なドキュメントを書く
- APIドキュメントを作成する
- コードコメントを生成する
- READMEファイルを維持する

### Debugging / デバッグ

- Analyze error messages and stack traces
- Identify root causes of issues
- Propose solutions and fixes
- Test fixes thoroughly
- エラーメッセージとスタックトレースを分析する
- 問題の根本原因を特定する
- 解決策と修正を提案する
- 修正を徹底的にテストする

### Refactoring / リファクタリング

- Improve code structure and organization
- Reduce code duplication
- Enhance readability and maintainability
- Preserve existing functionality
- コード構造と組織を改善する
- コードの重複を減らす
- 可読性と保守性を向上させる
- 既存の機能を保持する

## Communication Skills / コミュニケーションスキル

- **Clarity / 明確性**
  - Provide clear explanations
  - Use appropriate technical terminology
  - 明確な説明を提供する
  - 適切な技術用語を使用する

- **Collaboration / 協力**
  - Work effectively with developers
  - Understand and clarify requirements
  - Provide helpful suggestions
  - 開発者と効果的に協力する
  - 要件を理解し明確化する
  - 有用な提案を提供する

- **Adaptability / 適応性**
  - Adjust to different coding styles
  - Learn project-specific conventions
  - Handle multiple programming languages
  - 異なるコーディングスタイルに適応する
  - プロジェクト固有の規約を学ぶ
  - 複数のプログラミング言語を扱う

## Continuous Learning / 継続的学習

- Stay updated with latest technologies
- Learn from code reviews and feedback
- Improve based on project requirements
- 最新技術を常に把握する
- コードレビューとフィードバックから学ぶ
- プロジェクト要件に基づいて改善する
